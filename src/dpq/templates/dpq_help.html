Main functionality and permissions

Push Queue is an application designed to support and track push activities. It supports user authentication, multiple branches, push history and basic statistics. Push Queue is based on Django opensource web application framework and thus is extremely scalable and extendable.

The key idea of Push Queue application is to limit permissions for unauthorized users and to collect basic statistics about push activities.

There are four user categories implemented in the application:

  • Unauthorized user. Can view push queue, history and statistics.
  • Authorized user. In addition to unauthorized user's permissions is allowed to add pushes to the queue and to modify own records.
  • Tester on duty. In addition to authorized user's permissions is allowed to modify all push records.
  • Coordinator. Has all listed above permissions and also is allowed to perform administrative tasks such as creating branches etc.

This division allows to gain control and stability of push queue.


Register new account

In order to be able to create push records in queue developer or tester shoud register an account. Click [Get account] link on the right side of the menu bar and provide desired username and password (see image below). Please do NOT use credentials from your corporate accounts!

After registering account user is able to login to the application. Click [Sign in] link on the right side of the menu bar and provide credentials.


Place push to queue

Registered users are allowed to place a record to the push queue. To do that, click on [Add] button and fill in all fields on pop-up window (see image below).

Next time you click [Add] button all fields except Description will be automatically populated from your previous push record. If desired branch is not listed in dropdown list, ask your coordinator to create it.


Modify existing push

Registered users are allowed to modify own records in the push queue. To modify record click on wrench icon in status column (see image below).

In pop-up window modify desired fields and click [Apply] button (see image below). Quick tip: if you want to move item in the queue, it is better to do that on All tab.

Note that users are allowed to modify only own records. Tester on duty is able to modify all records with statuses waiting and in progress. If you have changed record status to done or reverted by mistake, please ask coordinator or tester on duty to change the status to previous value.


Administrative tasks

Coordinators and tester on duty have access to administration page (see image below). It allows to manage registered users, branches, push records and also statistics records.
